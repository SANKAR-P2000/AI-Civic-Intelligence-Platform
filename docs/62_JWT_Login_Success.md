# JWT Login Success

## Objective

Verify that the user can successfully authenticate using the login API and receive a valid JSON Web Token (JWT) for accessing protected resources.

---

## Endpoint

**POST**

```
/api/users/login
```

---

## Request Headers

```
Content-Type: application/json
```

---

## Request Body

```json
{
  "email": "anand@gmail.com",
  "password": "password123"
}
```

---

## Successful Response

**HTTP Status**

```
200 OK
```

**Response Body**

```json
{
  "id": 3,
  "fullName": "Anand Kumar",
  "email": "anand@gmail.com",
  "phoneNumber": "9876543212",
  "role": "CITIZEN",
  "loginTime": "2026-07-20T19:58:20.4307773",
  "token": "eyJhbGciOiJIUzI1NiJ9..."
}
```

---

## Response Fields

| Field       | Description                                      |
| ----------- | ------------------------------------------------ |
| id          | Unique user ID                                   |
| fullName    | User's full name                                 |
| email       | Registered email address                         |
| phoneNumber | User's phone number                              |
| role        | User role (CITIZEN, ADMIN, NGO, etc.)            |
| loginTime   | Login timestamp generated by the server          |
| token       | JWT access token used for authenticated requests |

---

## Authentication Flow

```
User enters Email + Password
            │
            ▼
Login API receives request
            │
            ▼
Find user by email
            │
            ▼
Verify password using BCrypt
            │
            ▼
Generate JWT Token
            │
            ▼
Return User Details + JWT Token
```

---

## JWT Token

The generated JWT contains authenticated user information and is digitally signed using the application's secret key.

Example

```
eyJhbGciOiJIUzI1NiJ9...
```

The token is used to access protected REST APIs.

---

## Using JWT in Future Requests

For protected endpoints, include the JWT in the HTTP Authorization header.

Example

```
Authorization: Bearer <JWT_TOKEN>
```

Example

```
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

## Security Notes

- Passwords are never stored in plain text.
- BCrypt is used to securely hash passwords before storing them.
- Original passwords cannot be recovered from BCrypt hashes.
- JWT tokens are digitally signed using the application's secret key.
- Clients must include the JWT token in the Authorization header to access protected endpoints.
- Authentication and authorization are separated, improving application security.
